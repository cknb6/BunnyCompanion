# Windows 发布前检查清单

## 1. 构建环境

- Windows 10 或 Windows 11，64 位。
- 安装 .NET 8 SDK，或 Visual Studio 2022 的“.NET 桌面开发”工作负载。
- 解压完整工程后再构建，不要直接在 ZIP 内运行脚本。

## 2. 一键发布

双击 `一键构建Windows版.bat`。脚本会完成清理、还原、Release 发布、单文件检查、中文文件名复制和 SHA-256 生成。

成功后只需使用：

```text
可直接发送/
├─ 小申陪伴.exe
├─ 使用说明.txt
└─ 版本校验.txt
```

不要把 `Artwork`、源码或参考素材发送给最终使用者。

## 3. 本机测试

- 双击 EXE，确认只出现一个桌宠实例。
- 检查待机、眨眼、走路、摸头、双击比心、拖动和托盘动作。
- 点击人物周围透明区域，确认不会挡住桌面图标或其他窗口。
- 打开设置页并保存一次，确认 `%LocalAppData%\BunnyCompanion\settings.json` 正常生成。
- 取消并重新勾选开机启动，确认托盘菜单状态正确。
- 测试隐藏、显示、鼠标穿透、全屏自动隐藏和完全退出。
- 若有双显示器，拖到另一屏幕并测试边界与缩放。
- 重新启动 Windows，确认桌宠随当前用户启动。

## 4. 发送前

- 完全退出测试实例。
- 使用 `版本校验.txt` 核对 EXE 的 SHA-256。
- 将“可直接发送”文件夹压缩后发送，避免聊天软件改名或拦截 EXE。
- 个人制作且未购买代码签名证书时，SmartScreen 可能提示未知发布者，使用说明中已给出解释。

## 5. 公开发布限制

本项目按私人礼物设计。若公开发布或商业化，应先获得参考照片人物的肖像授权，并使用可信代码签名证书重新签名发布文件。
